# Папка `simulations` — результаты прогонов

После каждого **завершённого run** (например, после `/run/beamOn` в конце симуляции) программа сохраняет сюда текстовый отчёт и CSV со спектром.

## Разные прогоны — подкаталог по тегу

В **начале макроса** (до `/run/beamOn`) можно задать имя подпапки:

```text
/sim/outputTag имя_прогона
```

Тогда файлы пишутся в **`simulations/имя_прогона/`** (например, `simulations/baseline_50k/run_summary.txt`). Так можно разделять отчёты по сценариям, не меняя код.

- Разрешены **латинские буквы, цифры, `_` и `-`**; остальные символы заменяются на `_`, длина обрезается.
- **`/sim/outputTag`** без аргумента или пустая строка — снова запись прямо в **`simulations/`** (как раньше).
- Тег задаётся **на процесс**: следующий прогон в том же сеансе использует последний выставленный тег, пока не смените командой.

## Файлы

| Путь | Содержание |
|------|------------|
| **`simulations/run_summary.txt`** | Итог прогона, если тег **не** задан. |
| **`simulations/<тег>/run_summary.txt`** | Итог, если задан **`/sim/outputTag <тег>`**. |
| **`spectrum_passed_habitat.csv`** (рядом с `run_summary`) | Гистограмма кинетической энергии частиц **в момент первого пересечения границы** жилого модуля (любая частица). Ось: логарифмические корзины по энергии (МэВ), столбец `count` — число частиц. |

Запускайте исполняемый файл из корня проекта **`Lera`** (рабочая директория), чтобы файлы появлялись в `Lera/simulations/`.

## Расшифровка блока «Run summary»

### Events

Число смоделированных **событий** (один первичный космический луч на событие, если не меняли генератор).

### Primary aimed at habitat (finite cylinder)

Сколько **первичных** частиц считаются **направленными на жилой модуль** в геометрическом смысле: луч из точки рождения вдоль начального импульса **пересекает** конечный цилиндр жилого модуля (радиус **2 м**, полудлина по **Z** **5 м**, включая торцевые крышки).

Это **не** «все запущенные частицы», а только те, чья траектория **целилась** в цилиндр по построению луча.

### … entered habitat

Из числа «aimed» сколько **первичных** треков **реально вошли** в объём жилого модуля (первое пересечение границы внутрь).

### Shielding efficiency (aimed primary, %)

**Эффективность защиты по направленным первичным** (в процентах):

```text
Eff = 100 * (N_aimed - N_entered_primary) / N_aimed
```

где **N_aimed** — число первичных с лучом на жилой модуль, **N_entered_primary** — сколько из них реально вошли в объём.

Интерпретация: среди первичных, которые **геометрически** летели в жилой модуль, какая доля **не попала** внутрь объёма (магнитное отклонение, торможение в экране и т.д.). Чем выше процент, тем **лучше** активная защита **для этой подвыборки**.

Если `Primary aimed` равен нулю, этот показатель неинформативен (деление не используется, в отчёте будет 0).

### Primary aimed (charged only) / … entered / Shielding efficiency charged

То же самое, но только для **заряженных** первичных (протон, α, Fe, e⁻; **без** γ). Отделяет **активную** (заряженную) компоненту от нейтральных первичных.

### Events with n/γ (track≠1) entering habitat (% of events)

**Процент событий**, в которых **хотя бы один вторичный** нейтрон или гамма-квант (номер трека **≠ 1**) **вошёл** в жилой модуль. Это ориентир на **пассивную** составляющую и **нейтральное / вторичное** излучение, а не на первичный пучок.

### Absorbed dose in habitat (air proxy), Gy

Суммарная **поглощённая энергия** в объёме жилого модуля (материал **воздух NIST**), делённая на **массу** этого цилиндра → **доза в Грэях (Гр)** для **воздуха** внутри «жилой полости». Это **не** доза в ткани человека.

### Equivalent dose estimate (Q=…), Sv

Грубая оценка **эквивалентной дозы**: `Gy × Q`, по умолчанию **Q = 2.5** (смешанное излучение; в коде можно менять `RunAction::SetQualityFactor`). Условная величина для порядка величины, не замена полноценной оценки риска.

### Neutron captures in MgB2 (count)

Число **нейтронов**, у которых конец трека — **захват** в материале **MgB₂** (соленоид / кольца). Зависит от физического списка (желательно **HP** для нейтронов) и статистики; при малом числе событий может быть **0**.

### Total track entries to habitat (diagnostic)

Сколько раз **любой трек** впервые пересёк границу **внутрь** жилого модуля (включая вторичные частицы и разные треки в одном событии). **Не** то же самое, что число событий.

### Events with any habitat entry

Сколько **событий**, в которых **хоть что-то** (первичное или вторичное) вошло в жилой модуль.

## Как получить отчёт

1. Соберите проект и запустите из каталога `Lera`, например:  
   `.\build\Release\MyShield.exe macros\run_stats.mac`
2. Откройте **`simulations/run_summary.txt`** и при необходимости **`simulations/spectrum_passed_habitat.csv`**.

Подробности источника частиц и макросов см. в **`README.md`** в корне проекта `Lera`.

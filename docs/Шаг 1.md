# Шаг 1   
📁 1. Структура проекта и файлы  
  
Для начала, в отдельной папке для проекта (например, Geant4_Project) вам понадобятся следующие файлы:  
  
1. CMakeLists.txt: Это основной файл для системы сборки CMake. Он сообщает компилятору, какие файлы нужно собрать и с какими библиотеками их связать.  
2. DetectorConstruction.hh и DetectorConstruction.cc: Файлы с кодом на C++, в которых описана геометрия вашей установки. Вы уже предоставили их код.  
3. vis.mac: Файл с макросами Geant4, который управляет визуализацией: как и что отображать.  
  
🔧 2. Файл CMakeLists.txt  
  
Создайте файл с именем CMakeLists.txt (без расширения) и скопируйте в него следующий код:  
  
  
# Минимальная версия CMake  
cmake_minimum_required(VERSION 3.10)  
  
# Имя проекта  
project(MyShield)  
  
# Поиск пакета Geant4 в системе  
find_package(Geant4 REQUIRED)  
  
# Добавление исполняемого файла, который будет собран из исходников  
add_executable(MyShield MyShield.cc DetectorConstruction.cc)  
  
# Подключение необходимых для Geant4 заголовочных файлов и библиотек  
target_link_libraries(MyShield ${Geant4_LIBRARIES})  
target_include_directories(MyShield PRIVATE ${Geant4_INCLUDE_DIRS})  
  
# Определение макроса для экспорта всех виртуальных таблиц Geant4  
add_definitions(-DG4MULTITHREADED)  
  
  
Этот файл говорит системе, как собрать ваш проект.  
  
⚙️ 3. Файл MyShield.cc (главный файл)  
  
Это главный исполняемый файл, который запускает симуляцию. Он нужен для вызова всех написанных вами классов. Для визуализации его можно оставить максимально простым.  
  
Создайте файл MyShield.cc:  
  
  
#include "G4RunManager.hh"  
#include "G4UImanager.hh"  
#include "G4VisExecutive.hh"  
#include "G4UIExecutive.hh"  
#include "DetectorConstruction.hh"  
  
int main(int argc, char** argv)  
{  
    // Менеджер запуска (ядро Geant4)  
    G4RunManager* runManager = new G4RunManager();  
  
    // Передаём в него вашу геометрию  
    runManager->SetUserInitialization(new DetectorConstruction());  
  
    // Инициализируем  
    runManager->Initialize();  
  
    // Создаём менеджер визуализации  
    G4VisManager* visManager = new G4VisExecutive();  
    visManager->Initialize();  
  
    // Создаём сессию для UI и передаём аргументы (например, для запуска макроса)  
    G4UIExecutive* ui = new G4UIExecutive(argc, argv);  
    G4UImanager::GetUIpointer()->ApplyCommand("/control/execute vis.mac");  
    ui->SessionStart();  
  
    // Чистим за собой  
    delete ui;  
    delete visManager;  
    delete runManager;  
    return 0;  
}  
  
  
Этот код создаёт ядро Geant4, инициализирует вашу геометрию, запускает визуализацию и выполняет команды из vis.mac.  
  
🎨 4. Файл vis.mac  
  
Теперь самый важный для этого шага файл — vis.mac. Он отвечает за внешний вид вашей геометрии.  
  
Создайте файл vis.mac в той же папке:  
  
  
# --- Настройка визуализации ---  
# Открыть визуализатор OpenGL (OGL)  
/vis/open OGL  
  
# --- Создание и настройка сцены ---  
# Отключить авто-обновление для ускорения  
/vis/viewer/set/autoRefresh false  
# Сделать так, чтобы меньше сообщений от визуализации засоряли консоль  
/vis/verbose errors  
  
# Отрисовать всю геометрию (ваш соленоид, кольца, корпус)  
/vis/drawVolume  
  
# Добавить оси координат (X-красный, Y-зелёный, Z-синий)  
/vis/scene/add/axes  
  
# Добавить простую шкалу масштаба  
/vis/scene/add/scale  
  
# Добавить номер события  
/vis/scene/add/eventID  
  
# Добавить логотип Geant4 (опционально)  
/vis/scene/add/logo  
  
# --- Настройка вида ---  
# Включить авто-обновление обратно  
/vis/viewer/set/autoRefresh true  
  
# Установить угол обзора (Theta=90, Phi=0 - вид сбоку по X)  
# Поиграйте с этими цифрами, чтобы найти лучший ракурс  
/vis/viewer/set/viewpointThetaPhi 90 0  
  
# Приблизить (1.0 = без зума)  
/vis/viewer/zoom 1.2  
  
# --- Для траекторий частиц (пока не нужно, но пригодится) ---  
# Включить сохранение траекторий (пока закомментируем, потом раскомментируете)  
# /tracking/storeTrajectory 1  
# Добавить траектории на сцену  
# /vis/scene/add/trajectories  
  
  
Этот макрос загрузит вашу геометрию, нарисует её с помощью OpenGL и покажет вам в отдельном окне.  
  
🖥️ 5. Пошаговая инструкция для запуска  
  
Откройте терминал и перейдите в вашу рабочую папку.  
  
1. Создайте папку для сборки: Всегда лучше собирать проект вне папки с исходниками, чтобы их не засорять. Выполните в терминале:  
   ```bash  
   mkdir build  
   cd build  
   ```  
2. Сконфигурируйте и соберите проект: Выполните следующие команды.  
   ```bash  
   cmake ../  
   make  
   ```  
   Если всё прошло успешно, в папке build появится исполняемый файл MyShield.  
3. Запустите визуализацию: Выполните следующую команду, чтобы запустить скомпилированную программу и увидеть вашу геометрию.  
   ```bash  
   ./MyShield  
   ```  
   Если всё сделано правильно, через несколько секунд откроется окно с трёхмерным изображением вашего соленоида, колец и жилого модуля.  
  
💡 Совет на будущее  
  
Попробуйте разные значения углов в файле vis.mac, например /vis/viewer/set/viewpointThetaPhi 45 45 (вид под углом) или /vis/viewer/set/viewpointThetaPhi 0 0 (вид сверху), чтобы лучше понять геометрию и расположение ваших компонентов. Удачи с первым шагом!
